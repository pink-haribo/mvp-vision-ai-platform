# Log Session Command

현재 대화 세션의 중요 내용을 타임라인 형식으로 정리하여 `docs/CONVERSATION_LOG.md` 파일에 기록합니다.

## 작업 순서

1. **현재 대화 분석**
   - 이 세션에서 논의된 주요 주제 파악
   - 중요한 결정사항 식별
   - 기술적 논의 내용 요약
   - 구현된 기능 정리

2. **타임라인 작성**
   - 날짜와 시간 기록
   - 논의 흐름을 시간순으로 정리
   - 핵심 결정사항 강조
   - 관련 파일/코드 참조 포함

3. **파일 업데이트**
   - `docs/CONVERSATION_LOG.md` 파일 읽기
   - 새 세션 내용을 **상단에** 추가 (최신 세션이 위로)
   - 기존 내용 유지
   - 파일이 없으면 새로 생성

## 타임라인 형식

```markdown
## [YYYY-MM-DD HH:MM] 세션 제목

### 논의 주제
- 주제 1
- 주제 2

### 주요 결정사항
1. **결정 제목**
   - 배경: ...
   - 결정: ...
   - 이유: ...

### 구현 내용
- 파일: `path/to/file.ts`
  - 변경사항: ...

### 다음 단계
- [ ] TODO 1
- [ ] TODO 2

### 관련 문서
- [설계 문서](../path/to/doc.md)

---
```

## 주의사항

- **간결하게**: 핵심만 기록 (상세 내용은 별도 문서에)
- **컨텍스트 제공**: 다음 세션에서 이해할 수 있도록
- **링크 활용**: 관련 파일, 문서, 코드 위치 참조
- **중복 방지**: 이미 문서화된 내용은 참조만 (전체 복사 X)

## 실행

이 명령어는 사용자가 `/log-session`을 입력하면 실행됩니다.
대화 내용을 자동으로 분석하여 타임라인을 생성하고 파일에 추가합니다.
